The Schema

        CREATE TABLE weather (
           id SERIAL,
           Station VARCHAR(32),
           StationName VARCHAR(64),
           Elevation DECIMAL(5,2),
           Latitude DECIMAL(7,4),
           Longitude DECIMAL(7,4),
           Date DATETIME,
           SkyConditions VARCHAR(32),
           Visibility DECIMAL(4,2),
           PresentWeatherType VARCHAR(32),
           DryBulbTempF INTEGER,
           WetBulbTempF INTEGER,
           DewPointTempF INTEGER,
           RelativeHumidity INTEGER,
           WindSpeed INTEGER,
           WindDirection INTEGER,
           WindGustSpeed INTEGER,
           StationPressure DECIMAL(4,2),
           PressureTendency INTEGER,
           PressureChange DECIMAL(4,2),
           SeaLevelPressure DECIMAL(4,2),
           Precipitation INTEGER,
           AltimeterSetting DECIMAL(4,2)
        );

This table was constructed by taking the CSV spreadsheet of the data and converting each column into the appropriate data type as specified by the spreadsheet documentation, or by looking at the values present within the data. These labeled columns were then used to create the above SQL instruction. Most of the numerical values use a fixed point decimal since an arbitrary level of precision is not necessary in many of these fields. Varchar is still used in places where the constraints are somewhat unknown such as the Station and StationName. SkyConditions and PresentWeatherType could be converted into an integer for a bitwise boolean array, but are not due to lack of a real need to save on space for this project. All the fields were left NULLable since there were many spots in the data with simply no value.
